# mrbgem-pn532 Implementation Plan (Phase 1a)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** PicoRuby/mruby から PN532 NFC リーダを I2C 経由で操作し、FeliCa IDm / MIFARE Type A UID を取得する薄いドライバ `mrbgem-pn532` を、host Ruby で完結する単体テスト付きで実装する。

**Architecture:** PN532 のフレームプロトコル (NXP UM0701-02) を純粋関数 `Frame.build` / `Frame.parse` に切り出し、I2C トランスポートは duck-typed なオブジェクトとして外部注入する。これにより MockI2C で完全に host 完結のテストが可能になる。コマンドは v1 で `GetFirmwareVersion (0x02)`, `InListPassiveTarget (0x4A)` の 2 種のみに絞る。

**Tech Stack:** Ruby (host: 3.3+ / target: PicoRuby), minitest, rake, GitHub Actions。設計書 §4.1 / §8 を入力とする。

## PN532 プロトコル早見表 (実装で参照)

### Information Frame (host → PN532)

```
00 00 FF LEN LCS TFI(0xD4) PD0(cmd) PD1..PDn DCS 00
```
- `LEN` = `1 + len(PD0..PDn)` (TFI + Data の合計バイト数)
- `LCS` = `(0x100 - LEN) & 0xFF`
- `DCS` = `(0x100 - (TFI + sum(PD0..PDn))) & 0xFF`
- TFI host→PN532 = `0xD4`、PN532→host = `0xD5`

### ACK Frame (PN532 → host)

```
00 00 FF 00 FF 00
```

### NACK Frame (host → PN532, 再送要求)

```
00 00 FF FF 00 00
```

### Error Frame (PN532 → host)

```
00 00 FF 01 FF 7F 81 00
```

### I2C Read 時の Ready Status バイト

I2C で PN532 から読み出すと、最初の 1 バイトが `ready status` で、`bit0 = 1` のときデータ準備完了。`bit0 = 0` ならまだ。1ms 程度の間隔でポーリングする。

### コマンドコード

| コマンド | code | 応答 code |
|---|---|---|
| GetFirmwareVersion | 0x02 | 0x03 (= cmd + 1) |
| InListPassiveTarget | 0x4A | 0x4B |

### InListPassiveTarget 引数

- `MaxTg`: 0x01 固定 (1 タグだけ)
- `BrTy`: 0x00 = 106kbps Type A (MIFARE), 0x01 = 212kbps FeliCa
- `InitiatorData` (FeliCa の場合): `00 FF FF 01 00` (System=FFFF, ReqCode=01, TimeSlots=00)

### InListPassiveTarget 応答

FeliCa 成功時:
```
D5 4B 01 01 LEN  (ここから POL_RES) IDm(8) PMm(8) [SystemCode(2)]
```
- `IDm` の 8 byte がカード固有 ID

Type A 成功時:
```
D5 4B 01 01 SENS_RES(2) SEL_RES NFCIDLen NFCID(4-10) [ATSLen ATS]
```
- `NFCID` の `NFCIDLen` byte 分が UID

検出失敗:
```
D5 4B 00
```

# mrbgems/mrbgem-pn532/src/pn532/frame.rb
require "pn532/errors"

class PN532
  module Frame
    PREAMBLE  = 0x00
    START1    = 0x00
    START2    = 0xFF
    POSTAMBLE = 0x00
    TFI_HOST_TO_PN532 = 0xD4
    TFI_PN532_TO_HOST = 0xD5

    ACK_BODY  = [0x00, 0xFF].freeze  # LEN, LCS
    NACK_BODY = [0xFF, 0x00].freeze

    def self.build(cmd, data)
      payload = [TFI_HOST_TO_PN532, cmd, *data]
      len = payload.size
      lcs = (0x100 - len) & 0xFF
      dcs = (0x100 - payload.sum) & 0xFF
      [PREAMBLE, START1, START2, len, lcs, *payload, dcs, POSTAMBLE]
    end

    # 受信バイト列を識別する。:ack / :nack / :error / :information
    # bytes の先頭にあるかもしれない 0x00 / 0xFF パディングは事前に剥がしてから渡される想定。
    def self.classify(bytes)
      raise ProtocolError, "frame too short: #{bytes.inspect}" if bytes.size < 5

      start = find_start(bytes)
      raise ProtocolError, "no start code (00 FF) in #{bytes.inspect}" unless start

      after_start = bytes[(start + 2)..]
      return :ack  if after_start[0, 2] == ACK_BODY
      return :nack if after_start[0, 2] == NACK_BODY

      # Error frame: LEN=0x01 LCS=0xFF TFI=0x7F (sum=80 → DCS=0x81)
      return :error if after_start[0, 4] == [0x01, 0xFF, 0x7F, 0x81]

      :information
    end

    # Information frame 専用。cmd と data を返す。
    # 失敗時は ChecksumError / ProtocolError を上げる。
    def self.parse_response(bytes)
      start = find_start(bytes)
      raise ProtocolError, "no start code in #{bytes.inspect}" unless start

      header = bytes[(start + 2)..]
      len = header[0]
      lcs = header[1]
      raise ChecksumError, "LCS mismatch: len=#{len.inspect} lcs=#{lcs.inspect}" \
        unless ((len + lcs) & 0xFF) == 0

      payload = header[2, len]
      dcs = header[2 + len]
      raise ChecksumError, "DCS mismatch" unless ((payload.sum + dcs) & 0xFF) == 0

      tfi = payload[0]
      raise ProtocolError, "unexpected TFI: 0x#{tfi.to_s(16)}" unless tfi == TFI_PN532_TO_HOST

      cmd  = payload[1]
      data = payload[2..] || []
      [cmd, data]
    end

    def self.find_start(bytes)
      bytes.each_cons(2).with_index do |(a, b), i|
        return i if a == START1 && b == START2
      end
      nil
    end
  end
end
```

class MockI2C
  attr_reader :write_log

  def initialize
    @write_log = []
    @read_queue = []
  end

  def write(addr, bytes)
    @write_log << [addr, bytes.dup]
    nil
  end

  def read(addr, len)
    raise "MockI2C: read called with empty response queue (addr=#{addr.inspect}, len=#{len})" if @read_queue.empty?
    response = @read_queue.shift
    response.first(len)
  end

  def queue_response(bytes)
    @read_queue << bytes.dup
  end
end
```

## Task 5: PN532#firmware_version

PN532 に GetFirmwareVersion (0x02) を投げ、ACK を受けてから応答を読み、Hash で返す。これが最も単純な往復で、I2C ready-status ポーリングを含めた骨格全体を確立する。

**Files:**
- Modify: `mrbgems/mrbgem-pn532/src/pn532.rb`
- Modify: `mrbgems/mrbgem-pn532/test/pn532_test.rb` (新規)

### 5.0 通信シーケンス (実装メモ)

```
host: write(addr, build_frame(0x02, []))
        → [00 00 FF 02 FE D4 02 2A 00]

host: poll for ack (read(addr, 7))   # 1 byte ready status + 6 byte ack frame
        ← 01 00 00 FF 00 FF 00       (ready=01, ACK)

host: poll for response (read(addr, N))
        ← 01 00 00 FF 06 FA D5 03 32 01 06 07 E8 00

host: drop leading ready byte → parse_response → cmd=0x03, data=[0x32, 0x01, 0x06, 0x07]
```

応答全長は `5 (preamble..len) + 1 (lcs) + LEN bytes + 1 (dcs) + 1 (postamble) + 1 (ready prefix)` = `LEN + 9`。GetFirmwareVersion は LEN=6 → 15 byte 読み出し。

PN532 が ready になるまでの polling は MockI2C 上では「最初の 1 バイトを読んで bit0 を見る」で再現する。本物相手では `time.sleep_ms(1)` を間に挟むが、host テストでは sleep を `Kernel#sleep` 経由でモックできるよう、`PN532#sleep_ms` を切り出して上書き可能にする。

```ruby
# mrbgems/mrbgem-pn532/src/pn532.rb
require "pn532/errors"
require "pn532/frame"

class PN532
  VERSION = "0.0.1"
  DEFAULT_ADDR = 0x24
  DEFAULT_TIMEOUT_MS = 1000
  POLL_INTERVAL_MS   = 1

  CMD_GET_FIRMWARE_VERSION  = 0x02
  CMD_INLIST_PASSIVE_TARGET = 0x4A

  def initialize(i2c:, addr: DEFAULT_ADDR)
    @i2c = i2c
    @addr = addr
  end

  def firmware_version(timeout_ms: DEFAULT_TIMEOUT_MS)
    cmd, data = round_trip(CMD_GET_FIRMWARE_VERSION, [], timeout_ms: timeout_ms)
    raise ProtocolError, "unexpected response cmd 0x#{cmd.to_s(16)}" unless cmd == CMD_GET_FIRMWARE_VERSION + 1
    raise ProtocolError, "firmware version data too short" if data.size < 4
    {ic: data[0], ver: data[1], rev: data[2], support: data[3]}
  end

  # Sleep を host テストで上書きできるよう切り出す。実機では Kernel.sleep を使う。
  def sleep_ms(ms)
    sleep(ms / 1000.0)
  end

  private

  # コマンドを送り、ACK と応答を待って (cmd_response, data) を返す
  def round_trip(cmd, data, timeout_ms:)
    @i2c.write(@addr, Frame.build(cmd, data))
    wait_for_ack(timeout_ms)
    read_response(timeout_ms)
  end

  def wait_for_ack(timeout_ms)
    bytes = poll_until_ready(7, timeout_ms)
    kind = Frame.classify(bytes)
    raise ProtocolError, "expected ACK, got #{kind}" unless kind == :ack
  end

  def read_response(timeout_ms)
    # 応答長は不明なので長めに 32 byte 読む。Frame.parse_response は start code を探すので余分は無害。
    bytes = poll_until_ready(32, timeout_ms)
    Frame.parse_response(bytes)
  end

  # I2C は最初の 1 byte に ready status を返す。bit0=1 になるまで待ち、
  # ready になったら追加分を読む。MockI2C では 1 回の read(len) で全部返ってくるため、
  # ここでは「先頭バイトの bit0 をチェックし、駄目なら次の応答キューを試す」という
  # 簡易プロトコルにする。実機向け実装は Phase 1b で I2C 仕様に合わせて見直す。
  def poll_until_ready(len, timeout_ms)
    elapsed = 0
    loop do
      raise TimeoutError, "PN532 did not become ready within #{timeout_ms}ms" if elapsed >= timeout_ms

      chunk = @i2c.read(@addr, len + 1) # +1 for ready status byte
      ready = chunk[0]
      if ready & 0x01 == 1
        return chunk[1..]
      end
      sleep_ms(POLL_INTERVAL_MS)
      elapsed += POLL_INTERVAL_MS
    end
  end
end
```

```ruby
  # MIFARE Type A UID を hex 文字列で返す。検出なし or タイムアウトは nil。
  def poll_typeA(timeout_ms: DEFAULT_TIMEOUT_MS)
    poll_passive_target(brty: 0x00, init_data: [], timeout_ms: timeout_ms) do |data|
      extract_typeA_uid(data)
    end
  end

  private

  # 共通: InListPassiveTarget を投げ、検出失敗/タイムアウトは nil、成功は yield 経由で抽出
  def poll_passive_target(brty:, init_data:, timeout_ms:)
    @i2c.write(@addr, Frame.build(CMD_INLIST_PASSIVE_TARGET, [0x01, brty, *init_data]))
    begin
      wait_for_ack(timeout_ms)
      cmd, data = read_response(timeout_ms)
    rescue TimeoutError
      return nil
    end
    raise ProtocolError, "unexpected response cmd 0x#{cmd.to_s(16)}" unless cmd == CMD_INLIST_PASSIVE_TARGET + 1
    return nil if data.first == 0  # NbTg = 0 → カード未検出
    yield data
  end

  # InListPassiveTarget Type A 応答 data の構造:
  #   NbTg(1) Tg(1) SENS_RES(2) SEL_RES(1) NFCIDLen(1) NFCID(NFCIDLen) [ATSLen ATS]
  # data[0]=NbTg, data[1]=Tg, data[2..3]=SENS_RES, data[4]=SEL_RES, data[5]=NFCIDLen
  def extract_typeA_uid(data)
    nfcid_len = data[5]
    raise ProtocolError, "invalid NFCID length: #{nfcid_len}" if nfcid_len.nil? || nfcid_len < 4 || nfcid_len > 10
    uid_bytes = data[6, nfcid_len]
    bytes_to_hex(uid_bytes)
  end

  def bytes_to_hex(bytes)
    bytes.map { |b| format("%02X", b) }.join
  end
```

(`private` が複数あっても問題ないが、見通しを保つため `wait_for_ack` 等の既存 private メソッドの直前に挿入する。実装者は適宜まとめてよい。)

  puts "FeliCa IDm: #{idm}"
elsif (uid = reader.poll_typeA(timeout_ms: 1000))
  puts "MIFARE UID: #{uid}"
else
  puts "no card"
end

## Phase 1a → 1b への申し送り

Phase 1b で扱うべき残課題 (最終レビューで列挙):

### PicoRuby/mruby 移植関連

1. **`Array#sum` の互換性確認**: `frame.rb` でフレーム payload の sum 計算に使用。mruby ではデフォルトで使えない可能性。`inject(0, :+)` への置換が安全。
2. **`define_singleton_method` の代替**: `pn532_test.rb` で `sleep_ms` をテスト時に no-op に上書きするのに使用。mruby は非サポート。Phase 1b ではテスト用サブクラスまたは依存注入 (`initialize` で `sleep_proc:` を受ける) パターンへ変更。
3. **endless range の置換**: `chunk[1..]` (`pn532.rb`) と `bytes[(start + 2)..]` (`frame.rb`) で使用。mruby のターゲットバージョンによっては未対応。`drop(1)` か `chunk[1, chunk.size - 1]` への置換が安全。
4. **PicoRuby の I2C API 確認**: 本計画では duck-type `i2c.write(addr, bytes)` / `i2c.read(addr, len)` を仮定。R2P2/picoruby-i2c の実 API と差があれば adapter を 1 枚噛ませる。
5. **mruby ホストビルド**: `mrbgem.rake` は Phase 1a で書いたが、`mrbtest` で実走させていない。Phase 1b で確認する。

### プロトコル・実機実装関連

6. **`poll_until_ready` の実機 I2C 仕様への適合**: 現在の実装は ready=0 のときも応答キューを 1 つ消費するモック特化の簡易プロトコル。実機では「ready ステータスバイトのみ読み、ready=1 になったら追加読み出し」の 2-pass 読み取りに作り直す必要がある。コードコメントにも明記済み。
7. **IRQ ピン対応**: 設計書では IRQ 経由 ready 判定がオプションと記載。v1 では polling のみ。Wokwi 結合時に必要なら拡張。
8. **実機タイミング**: 1ms ポーリング間隔は mock では 0ms に上書きしている。実機では PN532 仕様の最大 ACK 遅延 (~15ms) と Wi-Fi スタック割り込みを考慮。
9. **`read_response` の 32 byte 固定読み取り**: ATS を含む長い Type A 応答や SystemCode 込みの FeliCa 応答で切り詰められる可能性。Phase 1b で「ヘッダ読み → LEN 確定 → 本体読み」の 2-pass 化を検討。
10. **Extended frame (LEN=0xFF 0xFF) の未対応**: NXP UM0701-02 §6.2.1.1。v1 ユースケースでは不要だが将来コマンド追加時に必要になる可能性。

### テストカバレッジ

11. **エラー伝播テストの追加**: `poll_typeA` / `poll_felica` から `ChecksumError` と `ProtocolError` がそのまま伝播することを保証するテストを Phase 1b 冒頭で追加 (現在は暗黙の動作)。
12. **FeliCa ResCode 検証**: `extract_felica_idm` で `data[3] == 0x01` (ResCode) を検証していない。壊れた応答で誤った IDm を返すリスク。

### 設計上の改善余地 (任意)

13. **NACK エラーの分離**: 現在は ACK 以外を全て `ProtocolError "expected ACK, got nack/error/information"` で扱う。NACK は意味が異なるので `NackError` への分離を検討。
14. **`sleep_ms` の public 露出**: テスト override のため public にしているが、`initialize(sleep_proc:)` パターンでよりクリーンに DI できる。Phase 1b の I2C API 適合タイミングで合わせて見直し。
